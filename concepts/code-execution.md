# Code Execution

## The idea

Code is **instructions for a computer**. When you run code, the computer goes through your instructions one at a time, top to bottom. That's it. That's the whole thing.

Each line is a step. The computer doesn't skip ahead, doesn't guess what you meant, doesn't have feelings about it. It does step 1, then step 2, then step 3.

```python
print("hello")     # step 1: print hello
print("world")     # step 2: print world
print("!")         # step 3: print !
```

Output:
```
hello
world
!
```

If you put step 3 first, "!" comes first. The computer doesn't care that it looks weird.

## Why this matters

Lots of mysterious behavior is actually just "the steps ran in a different order than you thought." If something is wrong, the first question is **what order did the lines actually run in?**

## How to earn the badge — *I Know What Code Does*

- You start with an empty file.
- You wrote every line yourself.
- You ran it. It worked.
